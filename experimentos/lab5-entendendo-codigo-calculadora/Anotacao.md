1. Do todo para o Detalhe
Quando precisamos analisar o código de algum projeto, começamos a ler esse código do todo para o detalhe:
* ENTENDER O QUE CADA PARTE FAZ.
* NÃO COMO ELE FAZ.

2. Panorama de responsabilidade.
Deveremos saber o que cada classe faz, mesmo sem entender, a principio, a lógica

3. Executar o programa
* Observer um programa pela sua execução e em conjunto ler o código do ponto de entrada pode facilitar o entendimento do conjunto todo (Não das partes).

4. LER. NÃO SUPOR!
* Quando lemos um código, não é um exercício de imaginação louca, é pura leitura.

5. Ordem de análise:
* Classes menores e menos complexas primeiro.
Obs: Geralmente classes menores são as mais simples.
